# i18n Design: Full App Translation (en + pt-br)

## Problem

All user-facing strings in Arcane Poker are hardcoded in English. The Settings page already has a language toggle (`en` / `pt-br`) wired to `SettingsContext`, but toggling it does nothing because no translation mechanism exists.

## Approach

Lightweight custom `useTranslation()` hook + JSON locale files. No external libraries. The hook reads the `language` value from the existing `SettingsContext` and resolves translation keys from statically imported JSON.

---

## Architecture

### File structure

```
src/i18n/
  locales/
    en.json          # English strings (source of truth for keys)
    pt-br.json       # Portuguese (BR) translations
  types.ts           # TranslationKey union type derived from en.json
  useTranslation.ts  # The hook
  index.ts           # Re-exports
```

### JSON shape ŌĆö nested by feature area

```json
{
  "common": { "appTitle": "Arcane Poker", "playAgain": "Play Again" },
  "nav": { "home": "Home", "newGame": "New Game", "tutorial": "Tutorial", ... },
  "actions": { "fold": "Fold", "call": "Call", "raise": "Raise", ... },
  "settings": { "sound": "Sound", "music": "Music", ... },
  "gameOver": { "fortuneFavors": "Fortune Favors You!", ... },
  "table": { "revealArcana": "Reveal Arcana", ... },
  "rules": { "howToPlay": "How to Play", ... },
  "handRanks": { "straightFlush": "Straight Flush", ... },
  "arcana": { "0": { "name": "The Fool", "effect": "..." }, ... },
  "tutorial": { "round1": { "intro": { "title": "...", "body": "..." }, ... } }
}
```

### Type safety

Recursive mapped type generates a union of all valid dot-paths from `en.json`:

```typescript
type NestedPaths<T, Prefix extends string = ""> = T extends string
  ? Prefix
  : T extends Record<string, unknown>
    ? { [K in keyof T & string]: NestedPaths<T[K], Prefix extends "" ? K : `${Prefix}.${K}`> }[keyof T & string]
    : never;

export type TranslationKey = NestedPaths<typeof en>;
```

A compile-time assertion ensures `pt-br.json` matches `en.json`'s shape.

### Hook API

```typescript
const { t, language } = useTranslation();

t("common.playAgain")                    // ŌåÆ "Play Again"
t("actions.call", { amount: 50 })        // ŌåÆ "Call 50" (interpolation via {{amount}})
```

- Fallback chain: pt-br value ŌåÆ en value ŌåÆ raw key
- `console.warn` on missing keys in development
- `{{var}}` interpolation via simple regex replace

### Integration

No new Context or Provider needed. The hook calls `useSettings()` internally to read `language`. When `setLanguage()` triggers a re-render, all components using `useTranslation()` update automatically.

## Migration scope

### Tarot card data (`src/data/tarot.ts`)

This file contains structured objects (`fullName`, `tags`, `description`) for 56 standard + 22 arcana cards. Rather than flattening into JSON, create parallel locale files:

- Rename current file ŌåÆ `src/data/tarot.en.ts`
- Create `src/data/tarot.pt-br.ts`
- New `src/data/tarot.ts` exports a function accepting `language` parameter

Similarly for `src/data/getTarotInfo.ts` ŌĆö add `language` parameter, callers pass it from `useSettings()`.

### Tutorial narrations (`src/tutorial/tutorialScript.ts`)

Change `TutorialNarration` to use translation keys instead of inline strings:

```typescript
interface TutorialNarration {
  trigger: TutorialTrigger;
  titleKey: TranslationKey;   // was: title: string
  bodyKey: TranslationKey;    // was: body: string
  highlightCards?: CardHighlight[];
}
```

`TutorialNarrationContent.tsx` calls `t(narration.titleKey)` / `t(narration.bodyKey)`.

### Utility functions (`src/utils/cardUtils.ts`)

`actionLabel()` and `formatHandRank()` return hardcoded English. Two options:

- **Option A**: Remove them; callers use `t("actions.fold")` and `t("handRanks.twoPair")` directly.
- **Option B**: Make them accept a `t` function parameter.

**Choice: Option A** for `actionLabel` (callers translate directly), **Option B** for `formatHandRank` (add a `handRankKey()` mapping helper since rank strings are kebab-case and need mapping to translation keys).
