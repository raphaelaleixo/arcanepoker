Implementation Approaches

Three ways to add the Deal animation — each with different trade-offs.

A

DealtCard Wrapper ★ Recommended

Create a thin DealtCard wrapper component that adds the Drop & Bounce keyframe on mount. It accepts a dealIndex prop (0, 1, 2…) to compute stagger delay. PlayingCard is untouched — the animation lives entirely in the wrapper.

Pros

  • Zero changes to finalized Card components
  • Animation triggers naturally on React mount
  • Clean, reusable, easy to remove later

Cons

  • Extra component in the tree
B

sx Prop on PlayingCard

Pass the animation as an sx prop directly to each PlayingCard at the call sites in PlayerSeat and CommunityArea. No new component needed.

Pros

  • No new files
  • Minimal code change

Cons

  • Animation logic scattered across two files
  • Harder to adjust later
C

Newly-dealt flag in Game State

Track a newlyDealtCardIds set in the store. Cards in this set animate; others don't. A timeout clears the flag after the animation completes.

Pros

  • Most precise control over which cards animate
  • Handles edge cases (e.g. reconnects)

Cons

  • Store complexity for a purely visual concern
  • Timeout management is fragile
